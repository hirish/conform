Installation

Conform.js is packaged and installable using Bower. You can install it with the command bower install conformjs.

Alternatively, the source code is available from GitHub at /hirish/conform. Build it with gulp setup build.

QuickStart

Conform provides drop in replacements for React form and input components. <form> becomes <Conform.form> and <input> becomes <Conform.input>. Conform form components accept an identical set of props to React form components. Conform input components accept all props that React input components do, as well as two additional properties, validator and onValidate. 

<form onSubmit={@onSubmit} />
<Conform.form onSubmit={@onSubmit} />
<input
	type="text"
	valueLink={@linkState 'email'}
	placeholder="username@domain.com"
	/>
<Conform.input
	type="text"
	valueLink={@linkState 'email'}
	placeholder="username@domain.com"
	validator={Conform.Validators.Contact.Email()}
	onValidate={@onValidate 'email'}
	/>

A validator is the function that tests whether a value is valid. Several common types are included with Conform, or you can write your own. This is covered below.

The onValidate prop is called every time the input is validated (i.e. form submission and when the input is blurred). It has the form function onValidate (isValid, errorMsgs, value) {...}, and should be used to display errors to the user. An example onValidate function might be (in CoffeeScript): 

onValidate: (name) ->
	(isValid, errorMsgs, value) =>
		@state.errors[name] = if isValid then null else errorMsgs
		@setState errors: @state.errors

A <Conform.form> will not submit (via default HTML or onSubmit) unless all nested <Conform.input> components are valid.

Writing Your Own Validator

It is also possible to write custom validators. These have the following API: 

customValidator = (value) ->
		if valid then
			# An empty return value implies there was no problem with the input value.
			return
		else
			# It is also possible to return multiple error messages (as an array).
			return "Error message"

It is also possible to compose the supplied Conform validators using Conform.Validators.And([validators]) and Conform.Validators.Or([validators]).

Example - Registration

This example showcases a simple registration form. We demonstrate several included validators (e.g. email), composition of several validators (username) and a custom validator (confirm your password). It is also worth noting that Conform. is completely compatible with the React LinkState mixin, which further cuts down on boilerplate code.

errorIcon is a shim for a function which displays an icon/tooltip when an error is present. 

<Conform.form onSubmit={@onSubmit}>

	<label>Username</label>
	<Conform.input
		type="text"
		validator={[
			Conform.Validators.Required()
			Conform.Validators.Length(4)
			Conform.Validators.AlphaNumeric()
		]}
		onValidate={@onValidate 'username'}
		valueLink={@linkState 'username'}
		placeholder="This is a really really long placeholder"
		/>
	{errorIcon 'username'}

	<label>Email</label>
	<Conform.input
		placeholder="username@domain.com"
		type="text"
		validator={Conform.Validators.Contact.Email()}
		onValidate={@onValidate 'email'}
		valueLink={@linkState 'email'}
		/>
	{errorIcon 'email'}

	<label>Password</label>
	<Conform.input
		type="password"
		validator={Conform.Validators.Password()}
		onValidate={@onValidate 'password'}
		valueLink={@linkState 'password'}
		/>
	{errorIcon 'password'}

	<label>Confirm Your Password</label>
	<Conform.input
		type="password"
		validator={confirmValidator}
		onValidate={@onValidate 'confirm'}
		valueLink={@linkState 'confirm'}
		/>
	{errorIcon 'confirm'}

	<input type="submit" value="Submit" />

</Conform.form>
